L6 Data Link Layer — Services & Sublayers Theory ▾
🔑 One-Line Definition
Layer 2 of OSI. Sits between Physical (L1) and Network (L3). Packages raw bits into frames, adds addressing, does error detection, and controls medium access.
Two Sublayers (IEEE 802)

LLC — Logical Link Control (Upper)

  • Protocol Multiplexing — identifies which upper-layer protocol (IP, ARP) a frame belongs to using Service Access Points (SAPs)
  • Error Detection & Correction — CRC, ARQ, checksums; requests retransmission if corrupted
  • Flow Control — Stop-and-Wait & Sliding Window to prevent fast sender overwhelming slow receiver
  • Framing & Synchronization — marks start/end of frames so receiver knows boundaries

MAC — Media Access Control (Lower)

  • Physical Addressing — assigns 48-bit MAC addresses (e.g., 00:1A:2B:3C:4D:5E)
  • Channel Access Control — CSMA/CD, CSMA/CA, Token Passing to prevent collisions
  • Framing — converts data into frames (header + payload + trailer with CRC)
  • Error Detection at Frame Level — basic CRC at hardware level, quickly discards corrupted frames

Framing Methods
  • Character Count — first field specifies number of characters. Problem: if count field corrupted, all boundaries lost.
  • Character Stuffing — special chars DLE STX (start) and DLE ETX (end). If DLE appears in data, escape it by inserting DLE before it.
    Data: A DLE B → Frame: DLE STX A DLE DLE B DLE ETX
  • Bit Stuffing (HDLC) — frames delimited by flag 01111110. After 5 consecutive 1s in data, sender inserts a 0. Receiver removes it.
    Data: 01111110 11111011111110 01111110
    Sent: 01111110 111110 11111100 01111110

Types of Errors

Single-Bit Error

  • Only one bit flipped per data unit
  • Rare in modern high-speed networks
  • Easier to detect with parity bits
Example
Sent: 1001101  →  Received: 1001111
Bit 5 flipped: 0 → 1

Burst Error

  • Two or more consecutive bits altered
  • Much more common — channel disturbances hit groups of bits
  • Length = first corrupted bit to last (bits in between may be fine)
  • Requires stronger methods like CRC
Example
Sent: 1101001111  →  Received: 1100010111
Bits 3–7 corrupted (burst length = 5)
Key Terms
Dataword = original data before encoding (e.g., 1101)
Codeword = dataword + redundant bits (e.g., 11011)
L6 Error Detection Methods — VRC, LRC, Checksum, CRC Calculation ▾
① VRC — Vertical Redundancy Check (Parity Bit)

Add one extra bit to each data unit so the total number of 1s is even (even parity) or odd (odd parity).

Sender Steps

  1. Count the number of 1s in the data unit.
  2. Even parity: parity bit = 0 if count is already even; = 1 if odd. (Odd parity: reverse.)
  3. Transmit data + parity bit.

Receiver Steps

  1. Count 1s in received data + parity bit.
  2. If parity rule violated → error detected; otherwise → assume no error.
Even Parity Example
Data: 10110010  →  four 1s (even)  →  parity bit = 0
Transmitted: 10110010 0

✓ Strengths

  • Very simple & cheap
  • Detects odd number of bit errors (1, 3, 5…)
  • Common in low-cost serial links

✗ Weaknesses

  • Fails on even number of flips (2 bits cancel out)
  • Not reliable for burst errors
② LRC — Longitudinal Redundancy Check (2D Parity)

Arrange data as a block of rows. Compute parity for each column → the column parities form an extra LRC row appended after the data.

Example — 4×8 block, even parity
D1 = 11001010
D2 = 01011011
D3 = 11110000
D4 = 00011101

Column parity row (LRC) = 01111000
Transmit D1, D2, D3, D4 + LRC row.

✓ Strengths

  • Detects most burst errors & multiple errors affecting columns/rows
  • Can locate and correct a single-bit error (row ∩ column)

✗ Weaknesses

  • More overhead than VRC
  • Cannot correct arbitrary multi-bit errors
  • Some pathological patterns escape detection
③ Checksum (Ones'-Complement Sum)

Divide data into fixed-length words (16-bit in practice). Sum them using ones'-complement arithmetic. Complement the sum → this is the checksum sent with data.

Sender (16-bit words)

  1. Break data into 16-bit words.
  2. Add all words in binary. If carry beyond 16 bits, wrap it around (add carry to LSB — this is ones'-complement addition).
  3. Take bitwise NOT of the final sum → Checksum.
  4. Transmit data + checksum.

Receiver

  1. Sum all received 16-bit words including the checksum using ones'-complement.
  2. If result = all 1s (0xFFFF) → no error. Otherwise → error detected.
8-bit Demo
W1 = 01010101, W2 = 01100110
Sum = 10111011  →  Checksum = 01000100 (bitwise NOT)
Receiver: 01010101 + 01100110 + 01000100 = 11111111 ✓
Note on Carries
If adding two 16-bit words gives a 17-bit result, wrap the carry (add it back into the LSB) before complementing. This is specific to ones'-complement arithmetic used in IP/TCP/UDP headers.

✓ Strengths

  • Easy to implement in software
  • Detects single-bit & many multi-bit errors
  • Used in IP, TCP, UDP headers

✗ Weaknesses

  • Cannot detect if bits are swapped or complementary errors cancel out
  • Less robust than CRC
④ CRC — Cyclic Redundancy Check (Polynomial Division)

Treat the bit sequence as a binary polynomial. Divide by a fixed generator polynomial G (known to both sides). Remainder = CRC bits. Receiver divides received frame by same G; zero remainder = no error.

Key Facts
  • Generator of degree r yields r CRC bits.
  • Detects all burst errors of length ≤ r.
  • Detects all single-bit errors (if G has ≥ 2 nonzero terms).
  • Implemented efficiently in hardware via linear-feedback shift registers (LFSRs).
  • Used in: Ethernet, HDLC, PPP, USB, storage devices.

Sender Steps

  1. Let M = message bits. Choose generator G of degree r.
  2. Multiply M by x^r → append r zeros to M (this is the dividend).
  3. Divide M·x^r by G using mod-2 (XOR) division.
  4. Let R = remainder (r bits). Transmit T = M followed by R.

Receiver Steps

  1. Receive T' (may be corrupted).
  2. Divide T' by G using mod-2 division.
  3. If remainder = 0 → assume no error. If ≠ 0 → error detected.
Worked Example
M = 1101011011 (10 bits), G = 10011 (degree 4, so r = 4)
Step 1: Append 4 zeros → 11010110110000
Step 2: XOR-divide by 10011 → remainder R = 1110
Step 3: Transmitted frame T = 11010110111110
Receiver divides 11010110111110 by 10011 → remainder = 0000 ✓
Transmitted Frame T = M + R = 14 bits

L6 Error Correction — Hamming Code & Single-Bit Correction Calculation ▾
Two Approaches to Error Correction

FEC — Forward Error Correction

  • Sender adds redundant bits to let receiver self-correct without retransmission
  • Examples: Hamming code, Reed-Solomon
  • Used in: real-time video, satellite, memory
  • Drawback: bandwidth overhead

ARQ — Automatic Repeat Request

  • Receiver detects error and asks sender to retransmit
  • Examples: Stop-and-Wait, Go-Back-N, Selective Repeat
  • Used in: TCP, file transfers
  • Drawback: latency from retransmission
Hamming Code — Key Principles

Developed by Richard W. Hamming (1950). Can detect and correct single-bit errors and detect (not correct) double-bit errors.

Step 1 — How many parity bits do I need?

2r ≥ m + r + 1   where m = data bits, r = parity bits needed
Example: m = 4 data bits
Try r = 2: 2² = 4, need 4+2+1 = 7 → 4 ≥ 7? FALSE
Try r = 3: 2³ = 8, need 4+3+1 = 8 → 8 ≥ 8? TRUE ✓
So 3 parity bits, total codeword = 4+3 = 7 bits

Step 2 — Parity Bit Positions

Parity bits go at positions that are powers of 2: positions 1, 2, 4, 8, 16, …

Step 3 — Which bits does each parity bit cover? (Check-Skip Method)

Parity BitPositionPatternCovers positions
P11Check 1, skip 11, 3, 5, 7, 9, 11…
P22Check 2, skip 22, 3, 6, 7, 10, 11…
P44Check 4, skip 44, 5, 6, 7, 12, 13…
P88Check 8, skip 88, 9, 10, 11, 12, 13, 14, 15…

Data bits: D1=1, D2=0, D3=1, D4=1. Need 3 parity bits → 7-bit codeword.

Parity Calculations (Even Parity)
P1 covers pos 1,3,5,7 → values: P1,1,0,1 → sum of known = 1+0+1=2 (even) → P1 = 0
P2 covers pos 2,3,6,7 → values: P2,1,1,1 → sum of known = 1+1+1=3 (odd) → P2 = 1
P4 covers pos 4,5,6,7 → values: P4,0,1,1 → sum of known = 0+1+1=2 (even) → P4 = 0
Final codeword: 0 1 1 0 0 1 1

Error Detection & Correction

Suppose bit 5 flips during transmission → received: 0 1 1 0 1 1 1

Recalculate parities at receiver
P1 (1,3,5,7): 0+1+1+1 = 3 (odd) → Error! → P1 bit = 1
P2 (2,3,6,7): 1+1+1+1 = 4 (even) → OK → P2 bit = 0
P4 (4,5,6,7): 0+1+1+1 = 3 (odd) → Error! → P4 bit = 1

Error location = P4 P2 P1 in binary = 1 0 1 = 5
Flip bit 5 → corrected codeword: 0 1 1 0 0 1 1 ✓
Extract data bits (3,5,6,7): 1 0 1 1 = original 1011 recovered!
Single-Bit Error Correction — Summary
  • Detection: Parity checks identify whether an error exists.
  • Location: The combination of failed parity bits (in binary) gives the position.
  • Correction: Flip the bit at that position — original data restored.
L7 Error Recovery Protocols — Stop-and-Wait, Go-Back-N, Selective Repeat Protocol ▾
ARQ Concept
Automatic Repeat Request: receiver detects errors in frames & requests retransmission. Uses ACK (positive) and NAK (negative) acknowledgments, plus timeouts.
① Stop-and-Wait

Send 1 frame → wait for ACK → send next

  • Simplest ARQ
  • Window size = 1
  • ✓ Simple to implement
  • ✓ Reliable for small transfers
  • ✗ Very slow on high-latency links (sender idles while waiting)
② Go-Back-N

Send up to N frames; on error retransmit that frame + ALL subsequent

  • Window size = N
  • Receiver discards out-of-order frames
  • ✓ Higher throughput than S&W
  • ✓ Simpler receiver (no buffer)
  • ✗ Wastes bandwidth retransmitting correct frames
③ Selective Repeat

Send up to N frames; on error retransmit ONLY the bad frame

  • Receiver buffers out-of-order frames
  • Most efficient
  • ✓ Best bandwidth utilization
  • ✓ Efficient on noisy/high-latency links
  • ✗ Complex receiver (needs buffers for out-of-order frames)

L8 Multiple Access Protocols — TDMA, CDMA, CSMA/CD, CSMA/CA Protocol ▾

Multiple devices share the same communication medium. Multiple Access Protocols define how to share it efficiently without collisions.

① TDMA — Time Division Multiple Access

Available time is divided into discrete time slots. Each user gets one exclusive slot per frame.

How:Time → frames → each frame has N fixed slots; user i always uses slot i
Example:GSM cellular — frame = 4.615 ms, 8 time slots, each phone assigned one
✓ Pros:Predictable, orderly, easy to manage in low/moderate traffic
✗ Cons:Wasted bandwidth if a device has nothing to send in its slot; requires strict sync
② FDD — Frequency Division Duplex

Uplink and downlink use separate frequency bands → simultaneous two-way communication.

Example:3G/4G — uplink 1920–1980 MHz, downlink 2110–2170 MHz
✓ Pros:Full duplex simultaneously; less interference
✗ Cons:Needs wider spectrum; inefficient for asymmetric traffic
③ CDMA — Code Division Multiple Access

All users share the same frequency at the same time, each distinguished by a unique pseudorandom spreading code.

  1. User's data is multiplied by their unique spreading code (chip sequence).
  2. All coded signals transmitted together over same frequency band.
  3. Receiver applies same code to extract user's data from the mixed signal (others look like noise).
Example:3G (WCDMA) — multiple phones transmit same band, distinguished by codes
✓ Pros:High spectral efficiency; better privacy; robust against multipath fading
✗ Cons:Complex encoding/decoding; needs precise power control (near-far problem)
④ CSMA/CA — Carrier Sense Multiple Access with Collision Avoidance

Used in wireless networks (Wi-Fi / IEEE 802.11) where collision detection is impractical.

  1. Device senses the channel (carrier sense).
  2. If idle, it may proceed to send data.
  3. Optionally sends RTS (Request to Send); receives CTS (Clear to Send) — resolves hidden node problem.
  4. If channel busy, wait for a random backoff time before trying again.
Example:Wi-Fi — Laptop A: senses idle → sends RTS → gets CTS → sends data → gets ACK
✓ Pros:Works for wireless (can't detect collisions like wired); reduces collisions
✗ Cons:Backoff + RTS/CTS overhead; less efficient in congested networks
⑤ CSMA/CD — Carrier Sense Multiple Access with Collision Detection

Used in wired Ethernet (IEEE 802.3). Devices detect collisions while transmitting.

  1. Device listens to channel.
  2. If idle, begins transmitting.
  3. Simultaneously monitors for collisions.
  4. On collision: stop immediately, send jam signal.
  5. Both devices wait a random backoff period (exponential backoff), then retry.
Example:Ethernet — two computers transmit simultaneously → collision → both back off → retry
✓ Pros:Simple for wired; automatic collision resolution
✗ Cons:Inefficient under heavy traffic; not usable for wireless

L9 MAC Addressing & Ethernet Network ▾
MAC Address
  • Unique 48-bit (6-byte) hardware address assigned to every NIC (Network Interface Card)
  • Written in hexadecimal: 00:1A:2B:3C:4D:5E
  • First 24 bits = OUI (Organizationally Unique Identifier) — identifies manufacturer
  • Last 24 bits = unique identifier assigned by manufacturer
  • Operates at Layer 2; used only within a local network segment (IP handles global delivery)
Address TypeValuePurpose
UnicastAA:BB:CC:DD:EE:01Delivered to one specific device
BroadcastFF:FF:FF:FF:FF:FFDelivered to ALL devices in the LAN
Multicast01:00:5E:xx:xx:xxDelivered to a group of devices
Ethernet (IEEE 802.3)

Most widely deployed LAN technology. Operates at Layer 2 and Layer 1. Uses frames. Originally used CSMA/CD; modern switched Ethernet is full-duplex (no collisions).

Why minimum 64 bytes?
In CSMA/CD, a device must detect a collision before finishing transmission. The minimum frame size ensures the frame is still on the wire when a collision signal returns. (Based on max cable length & propagation delay.)
L9 Polling Protocol ▾

A centralized control technique where a master device sequentially asks (polls) each slave device whether it has data to send. Only the polled device may transmit.

How Polling Works
  1. Master sends a poll signal to slave A.
  2. Slave A responds only if it has data; otherwise, no response.
  3. Master then polls slave B, slave C, and so on.
  4. Cycle repeats continuously.
Types of Polling
TypeHowEfficiency
Roll Call PollingMaster polls ALL devices in fixed sequence regardless of data statusSimple but wastes time on idle devices
Hub / Selective PollingMaster only polls devices likely to have dataMore efficient; reduces idle wait
Interrupt PollingDevices raise a request flag; master responds to requestsMost efficient; no unnecessary polling

✓ Advantages

  • Orderly: only one device transmits at a time — no collisions
  • Simplified error handling (master controls access)
  • Predictable behavior — devices know when they'll be polled
  • Fairness — all devices get a turn

✗ Disadvantages

  • Delay — devices must wait for their poll turn
  • Overhead — polling messages consume bandwidth
  • Single point of failure — if master fails, network stops
  • Inefficient for idle devices

Wi-Fi (IEEE 802.11) — Key Points
  • Uses CSMA/CA (Collision Avoidance) — cannot detect collisions on wireless
  • Hidden node problem solved by RTS/CTS handshake
  • Uses OFDM (Orthogonal Frequency Division Multiplexing) for multicarrier transmission
  • Two modes: Infrastructure Mode (via Access Point) and Ad-hoc Mode (peer-to-peer)
  • WPA2/WPA3 for security; SSID for network identification
Bluetooth (IEEE 802.15.1) — Key Points
  • Designed for short-range Personal Area Networks (PAN)
  • Operates at 2.4 GHz using Frequency Hopping Spread Spectrum (FHSS) — hops between 79 channels 1600 times/sec → interference resistance
  • Master-slave architecture: one master, up to 7 active slaves in a piconet
  • Multiple piconets can overlap → scatternet
  • Bluetooth 5.x: up to 50 Mbps, range up to 240 m (outdoors)
  • Uses Time Division Duplex (TDD) for bidirectional communication
Ethernet (IEEE 802.3) — Key Points
  • Defined by IEEE 802.3 standards family
  • Traditional: CSMA/CD on shared bus/hub → modern: full-duplex on switched networks (no CSMA/CD needed)
  • Supports speeds from 10 Mbps → 100 Gbps and beyond
  • Frame format standardized — max payload 1500 bytes (MTU)
  • Auto-negotiation: devices agree on speed & duplex automatically

What are the 2 sublayers of the Data Link Layer?
LLC (Logical Link Control) — upper sublayer; handles protocol mux, error/flow control.
MAC (Media Access Control) — lower sublayer; handles physical addressing & channel access.

What is a Burst Error?
Two or more consecutive bits altered in a data unit. Length = first corrupted bit to last corrupted bit. Much more common than single-bit errors. Requires CRC to detect.
tap
VRC parity bit rule for even parity?
Count the 1s in the data. If count is even → parity bit = 0. If count is odd → parity bit = 1. After adding parity bit, total 1s should be even.
tap
What is the Hamming Code formula for parity bits?
2^r ≥ m + r + 1 where r = number of parity bits needed, m = number of data bits. Find smallest r that satisfies this inequality.
tap
Where are parity bits placed in Hamming Code?
At positions that are powers of 2: positions 1, 2, 4, 8, 16, … All other positions hold data bits.
tap
What does P1 (position 1) check in Hamming Code?
Check-Skip Method: Check 1, skip 1 → covers positions 1, 3, 5, 7, 9, 11… (all odd positions)
tap
How does Hamming Code locate an error?
The binary number formed by the failing parity checks (P4 P2 P1) gives the position of the erroneous bit. Flip that bit to correct the error.
tap
CRC: What happens at the receiver?
Divide received frame (T') by the same generator G using mod-2 division. If remainder = 0 → no error. If remainder ≠ 0 → error detected.
tap
What is the difference between Go-Back-N and Selective Repeat?
Go-Back-N: on error, retransmit the bad frame AND all subsequent frames. Receiver discards out-of-order frames.
Selective Repeat: on error, retransmit ONLY the bad frame. Receiver buffers out-of-order frames.
tap
CSMA/CA vs CSMA/CD — which is used where?
CSMA/CD → wired Ethernet (IEEE 802.3). Detects collisions during transmission.
CSMA/CA → wireless Wi-Fi (IEEE 802.11). Avoids collisions (can't detect on wireless). Uses RTS/CTS & backoff.
tap
MAC address format & size?
48-bit (6-byte) identifier written in hex, e.g. 00:1A:2B:3C:4D:5E. First 24 bits = OUI (manufacturer ID). Last 24 bits = device-specific ID. Broadcast = FF:FF:FF:FF:FF:FF.
tap
Ethernet frame: what is the FCS field?
Frame Check Sequence — 4 bytes. Contains CRC-32 checksum. Receiver recalculates CRC; if it matches FCS → frame is accepted. If not → discarded.
tap
Ethernet minimum frame size and why?
64 bytes minimum. Required so the frame is still being transmitted when a collision signal returns (based on max cable propagation delay). Ensures collision detection works properly in CSMA/CD.
tap
What is Bit Stuffing?
Used in HDLC. Frames delimited by 01111110. After 5 consecutive 1s in the data, sender inserts an extra 0 bit. Receiver strips the stuffed 0 after receiving 5 ones. Prevents data from mimicking the flag pattern.
tap
What is CDMA and how does it separate users?
Code Division Multiple Access. All users share the same frequency at the same time. Each user's data is multiplied by a unique spreading code. Receiver uses the same code to extract signal; other users' signals appear as noise.
tap
What is Polling and what are its types?
Centralized access control where a master sequentially asks slaves if they have data. Types: Roll Call (fixed order, all polled), Hub/Selective (likely-active only), Interrupt (slave requests polling).
tap
IEEE 802.11 uses which MAC protocol?
CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance). Collision detection is impractical on wireless because a transmitting device can't simultaneously hear collisions.
tap
What frequency hopping technique does Bluetooth use?
FHSS — Frequency Hopping Spread Spectrum. Hops between 79 channels at 2.4 GHz, 1600 times per second. Makes it resistant to interference and difficult to intercept.
tap
LRC can correct which type of error?
Single-bit errors only. The row with wrong parity and the column with wrong parity intersect at the error location. That bit is flipped to correct it. Multi-bit errors detected but not always correctable.
tap
Checksum sender: what is the final step?
After computing the ones'-complement sum of all data words (wrapping any carry), take the bitwise NOT (ones' complement) of the sum. This result is the checksum sent alongside the data.
tap
What does FEC stand for and when is it used?
Forward Error Correction. Sender adds enough redundant bits that receiver can self-correct without retransmission. Used when retransmission is unacceptable — real-time video, satellite, streaming, memory storage.
tap
What is the hidden node problem in Wi-Fi?
Two devices (A and C) are both in range of an access point (B) but out of range of each other. Both sense the channel as idle and transmit simultaneously → collision at B. Solved by RTS/CTS handshake in CSMA/CA.
