Data Link Layer Overview

OSI Layer 2 · LLC · MAC · Services

Definition

The Data Link Layer (Layer 2) provides reliable delivery of frames between directly connected nodes, bridging the Physical Layer and Network Layer.

Sublayers (IEEE 802)
LLC — Logical Link Control
Upper sublayer. Interface between Network Layer (IPv4, IPv6, ARP) and MAC. Handles protocol multiplexing via SAPs, flow control, and error detection.
MAC — Media Access Control
Lower sublayer. Handles physical transmission, 48-bit MAC addressing, channel access (CSMA/CD, CSMA/CA), and hardware-level CRC.

Error Detection Methods

VRC · LRC · Checksum · CRC

Error Types
Single-Bit Error
Only 1 bit altered. Rare in high-speed networks.
Burst Error
2+ consecutive bits altered. More common; requires CRC.
Causes: Thermal noise (background), Impulse noise (lightning/switching), Cross talk, Attenuation, Jitter
Error Detection Methods Comparison
MethodDetectsOverheadUsed In
VRC (Parity)Odd bit errors only1 bit/unitSerial links
LRC (2-D Parity)Burst + single-bit errors1 byte/blockBlock protocols
ChecksumSingle + many multi-bit16 bitsIP, TCP, UDP
CRCAll bursts ≤ r bitsr bitsEthernet, HDLC, USB
CRC (Cyclic Redundancy Check) — Detailed ▼ expand
Concept: Treat bit sequence as polynomial; divide by generator polynomial G; remainder R appended.
Sender:
1. Append r zeros to M
2. XOR-divide by G
3. R = remainder
4. Transmit T = M + R
Receiver:
Divide T by G
Remainder = 0 → OK ✅
Remainder ≠ 0 → Error ❌
Example: M=1101011011, G=10011 (degree 4)
Append 4 zeros → 11010110110000
XOR-divide by 10011 → R=1110
Transmit T=11010110111110
Exam Tip

CRC detects all burst errors of length ≤ r (where r = degree of generator polynomial). Cannot correct errors — only detect.

Unit 4 · 2 hrs

Error Correction

Hamming Code · FEC · ARQ Overview

Two Approaches
FEC
Forward Error Correction — redundant bits allow receiver to self-correct. Good for real-time/satellite (no retransmission delay).
ARQ
Automatic Repeat reQuest — receiver detects error → requests retransmission. Used in TCP. Reliable but has latency.
Hamming Code
Corrects single-bit errors; detects double-bit errors. Developed by R.W. Hamming (1950).
Formula: 2^r ≥ m + r + 1
where r = parity bits, m = data bits
Parity bit positions: Powers of 2 → 1, 2, 4, 8, ...
Check-Skip Method: P1 checks 1,3,5,7... (check 1, skip 1); P2 checks 2,3,6,7... (check 2, skip 2); P4 checks 4,5,6,7... (check 4, skip 4)
Example (m=4, data=1011):
Position: 1 2 3 4 5 6 7
Bit: P1 P2 1 P4 0 1 1
P1 = 0 (even parity), P2 = 1, P4 = 0
Codeword: 0 1 1 0 0 1 1
Error Correction Logic

Failed parity checks form binary number → error position. Flip that bit. Example: P1 fails (1), P2 passes (0), P4 fails (1) → binary 101 = position 5.

ARQ Error Recovery Protocols

ARQ Comparison
ProtocolWindowRetransmitBufferEfficiency
Stop-and-Wait11 frameNot neededLow
Go-Back-NNFrom error + all subsequentNot neededMedium
Selective RepeatNOnly erroneous frameRequiredHigh
Stop-and-Wait ARQ
Send 1 frame → wait for ACK/NAK/timeout → send next. Simple but inefficient; sender idle during wait.
Go-Back-N ARQ
Transmit up to N frames. On error in frame k: discard k and ALL subsequent. Retransmit k onwards. Wastes bandwidth.
Selective Repeat ARQ
Buffer out-of-order frames; request retransmission of ONLY erroneous frame. Best efficiency but complex.
Exam Tip

Go-Back-N: receiver discards all frames after error (no buffer needed). Selective Repeat: receiver buffers out-of-order frames (buffer required).

CSMA/CD — Collision Detection
• Sense channel → transmit if idle
• Monitor for collision during transmission
• On collision: stop, send jam signal, random backoff, retry
• Used in: Wired Ethernet
✗ Inefficient under high load; not suitable for wireless
CSMA/CA — Collision Avoidance
• Sense channel; if idle → transmit; if busy → wait random backoff
• Optional RTS/CTS handshake avoids hidden node problem
• Used in: Wi-Fi (802.11)
✓ Good for wireless
✗ Overhead from backoff/control frames
TDMA / FDD / CDMA Details ▼ expand
TDMA: Time divided into slots; each user gets exclusive slot. Used in GSM (8 slots per 4.615 ms frame). ✓ Orderly; ✗ Wasted slots if idle.
FDD: Uplink and downlink on separate frequency bands. Used in 3G/4G. ✓ Simultaneous two-way; ✗ Needs wider spectrum.
CDMA: All users share same frequency; distinguished by unique spreading codes. Used in 3G/WCDMA. ✓ High spectral efficiency; ✗ Complex, needs power control.
Polling
Concept: Centralized access control — master polls slave devices sequentially.
Types: Roll Call (polls all), Hub/Selective (polls likely-active), Interrupt (devices request permission)
Pros: Orderly, no collisions, predictable scheduling
Cons: Transmission delay, polling overhead, single point of failure

IEEE Standards — Wireless LANs & Bluetooth

802.11 · 802.15

IEEE 802.11 — Wireless LAN (Wi-Fi)
• Covers Physical Layer and Data Link Layer (MAC sublayer)
• Operates in 2.4 GHz ISM band (802.11b/g) or 5 GHz UNII band (802.11a)
• Uses CSMA/CA for medium access (not CSMA/CD — can't detect collisions in wireless)

802.11 Architecture
BSS (Basic Service Set): Building block of WLAN. Stations + optional AP.
Ad-hoc: BSS without AP; stand-alone network
Infrastructure: BSS with AP; connects to distribution system
ESS (Extended Service Set): Multiple BSSs connected via distribution system
802.11 MAC Functions
DCF (Distributed Coordination Function): CSMA/CA — mandatory
PCF (Point Coordination Function): Centralized polling — optional
Frame Types: Management (beacon, probe), Control (RTS/CTS/ACK), Data
IFS (Interframe Space): Priority timing: SIFS < PIFS < DIFS
IEEE 802.15 — Bluetooth (WPAN)
• Wireless Personal Area Network technology for short-range cable replacement
• Operates in 2.4 GHz ISM band (79 channels of 1 MHz each)
• Uses Frequency Hopping Spread Spectrum (FHSS) — hops 1600 times/second
• Range: 10m (Class 2), up to 100m (Class 1)
• Data rate: ~1 Mbps (Basic Rate), up to 3 Mbps (Enhanced Data Rate)

• Up to 8 stations (1 primary + 7 active secondaries)

• Multiple piconets linked together
• A secondary in one piconet can be primary in another
• Enables larger network coverage
Syllabus Note

IEEE Standards for Wireless LANs (802.11), Ethernet (802.3), and Bluetooth (802.15) are explicitly listed in the syllabus. 802.11 uses CSMA/CA (not CSMA/CD) because wireless cannot detect collisions while transmitting.
