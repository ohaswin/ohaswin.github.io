🗄️ RDBMS Lab Exam Guide

Quick revision for tomorrow's practical

1. Table with Constraints (PK, FK)

DDL

What: Rules to keep data clean and connected between tables.

-- Create table with Primary Key CREATE TABLE Student ( roll_no INT PRIMARY KEY, name VARCHAR(50), age INT ); -- Create table with Foreign Key CREATE TABLE Marks ( mark_id INT PRIMARY KEY, roll_no INT, subject VARCHAR(30), marks INT, FOREIGN KEY (roll_no) REFERENCES Student(roll_no) );
Real Life: Student table = master list of students. Marks table = their scores. FK ensures you can't add marks for a student who doesn't exist.
🔑 PK = Unique ID for each row. FK = Links to another table's PK.

2. DML Commands (INSERT, UPDATE, DELETE)

Data Manipulation

What: Add, change, or remove data.

-- INSERT: Add new data INSERT INTO Student VALUES (1, 'Rahul', 20); INSERT INTO Student (roll_no, name) VALUES (2, 'Priya'); -- UPDATE: Change existing data UPDATE Student SET age = 21 WHERE roll_no = 1; -- DELETE: Remove data DELETE FROM Student WHERE roll_no = 2;
Remember: Always use WHERE with UPDATE and DELETE, or you'll change/delete everything!

3. ALTER & DROP (DDL)

Structure Changes
-- ALTER: Modify existing table ALTER TABLE Student ADD email VARCHAR(50); ALTER TABLE Student DROP COLUMN age; ALTER TABLE Student MODIFY name VARCHAR(100); -- DROP: Delete entire table DROP TABLE Student;
⚠️ DROP deletes everything forever. ALTER just changes structure, keeps data.

4. ANY, ALL, EXISTS, NOT EXISTS

Subquery Operators
OperatorMeaning
ANYTrue if matches ANY value in list
ALLTrue if matches ALL values in list
EXISTSTrue if subquery returns ANY row
NOT EXISTSTrue if subquery returns NO rows
-- Find students with marks higher than ANY student in Class B SELECT * FROM Student WHERE marks > ANY (SELECT marks FROM Student WHERE class = 'B'); -- Find departments that have at least one employee SELECT dept_name FROM Department d WHERE EXISTS (SELECT 1 FROM Employee e WHERE e.dept_id = d.dept_id);

5. Set Operations (UNION, INTERSECT, EXCEPT)

Combining Results
-- UNION: Combine results (removes duplicates) SELECT name FROM ClassA UNION SELECT name FROM ClassB; -- UNION ALL: Combine (keeps duplicates) SELECT name FROM ClassA UNION ALL SELECT name FROM ClassB; -- INTERSECT: Common to both SELECT name FROM ClassA INTERSECT SELECT name FROM ClassB; -- EXCEPT/MINUS: In first but not second SELECT name FROM ClassA EXCEPT SELECT name FROM ClassB;
📋 Both queries must have same number of columns and compatible data types.

6. LIKE & BETWEEN

Pattern Matching
-- LIKE: Pattern matching SELECT * FROM Student WHERE name LIKE 'A%'; -- Starts with A SELECT * FROM Student WHERE name LIKE '%a'; -- Ends with a SELECT * FROM Student WHERE name LIKE '%am%'; -- Contains 'am' SELECT * FROM Student WHERE name LIKE 'R_hul'; -- R_hul (Rahul, Rihul) -- BETWEEN: Range check (inclusive) SELECT * FROM Student WHERE marks BETWEEN 60 AND 80; SELECT * FROM Orders WHERE date BETWEEN '2024-01-01' AND '2024-03-31';
Wildcards: % = any characters (0 or more), _ = exactly one character

7. Nested & Correlated Queries

Subqueries
-- NESTED: Inner query runs first, once SELECT * FROM Student WHERE marks > (SELECT AVG(marks) FROM Student); -- CORRELATED: Inner query runs for each outer row SELECT name FROM Student s1 WHERE marks > (SELECT AVG(marks) FROM Student s2 WHERE s2.class = s1.class);
Difference: Nested = Inner query runs once. Correlated = Inner query runs for every row in outer query (slower but powerful).

8. JOINS

Combining Tables
-- INNER JOIN: Only matching rows SELECT s.name, m.subject, m.marks FROM Student s INNER JOIN Marks m ON s.roll_no = m.roll_no; -- LEFT JOIN: All from left, matched from right SELECT s.name, m.marks FROM Student s LEFT JOIN Marks m ON s.roll_no = m.roll_no; -- RIGHT JOIN: All from right, matched from left SELECT s.name, m.marks FROM Student s RIGHT JOIN Marks m ON s.roll_no = m.roll_no; -- EQUI JOIN: Join using = operator (same as INNER JOIN) SELECT * FROM Student s, Marks m WHERE s.roll_no = m.roll_no;
Memory Trick: LEFT = Left table is boss (all rows kept). INNER = Only common friends kept.

9. Aggregate Functions

Calculations
-- Basic aggregates SELECT COUNT(*) FROM Student; -- Total rows SELECT COUNT(roll_no) FROM Student; -- Non-null roll_no SELECT SUM(marks) FROM Marks; -- Total marks SELECT AVG(marks) FROM Marks; -- Average marks SELECT MAX(marks) FROM Marks; -- Highest marks SELECT MIN(marks) FROM Marks; -- Lowest marks
🧮 These functions work on groups of rows, not individual rows.

10. GROUP BY & HAVING

Grouping Data
-- GROUP BY: Group rows by column SELECT class, COUNT(*) as total_students FROM Student GROUP BY class; -- HAVING: Filter groups (like WHERE but for groups) SELECT class, AVG(marks) as avg_marks FROM Student GROUP BY class HAVING AVG(marks) > 60; -- Combined with WHERE SELECT class, COUNT(*) as passed FROM Student WHERE marks >= 40 GROUP BY class HAVING COUNT(*) > 5;
WHERE vs HAVING: WHERE filters rows BEFORE grouping. HAVING filters groups AFTER aggregation.

11. VIEWS

Virtual Tables
-- CREATE VIEW: Save a query as virtual table CREATE VIEW TopStudents AS SELECT name, marks FROM Student WHERE marks > 80; -- Use the view SELECT * FROM TopStudents; -- UPDATE VIEW: Modify view definition CREATE OR REPLACE VIEW TopStudents AS SELECT name, marks, class FROM Student WHERE marks > 75; -- DELETE from view (affects base table) DELETE FROM TopStudents WHERE marks < 70; -- DROP VIEW: Remove view DROP VIEW TopStudents;
👁️ View doesn't store data, just the query. Data comes from original tables.

📝 Complete Practice Exercise

Scenario: A small Kirana Store database. Follow these questions step by step.

1 Create a Customers table with: cust_id (PK), name, phone, city.
CREATE TABLE Customers ( cust_id INT PRIMARY KEY, name VARCHAR(50), phone VARCHAR(15), city VARCHAR(30) );
2 Create an Orders table with: order_id (PK), cust_id (FK), amount, order_date.
CREATE TABLE Orders ( order_id INT PRIMARY KEY, cust_id INT, amount DECIMAL(10,2), order_date DATE, FOREIGN KEY (cust_id) REFERENCES Customers(cust_id) );
3 Insert 3 customers: (1, 'Ramesh', '9876543210', 'Delhi'), (2, 'Suresh', '9876543211', 'Mumbai'), (3, 'Priya', '9876543212', 'Delhi').
INSERT INTO Customers VALUES (1, 'Ramesh', '9876543210', 'Delhi'), (2, 'Suresh', '9876543211', 'Mumbai'), (3, 'Priya', '9876543212', 'Delhi');
4 Insert orders: (101, 1, 500, '2024-03-01'), (102, 1, 1200, '2024-03-15'), (103, 2, 800, '2024-03-10'), (104, 3, 1500, '2024-03-20').
INSERT INTO Orders VALUES (101, 1, 500, '2024-03-01'), (102, 1, 1200, '2024-03-15'), (103, 2, 800, '2024-03-10'), (104, 3, 1500, '2024-03-20');
5 Update Ramesh's phone to '9999999999'.
UPDATE Customers SET phone = '9999999999' WHERE name = 'Ramesh';
6 Add a column 'email' to Customers table.
ALTER TABLE Customers ADD email VARCHAR(50);
7 Show all customers from Delhi using LIKE (match 'Delhi').
SELECT * FROM Customers WHERE city LIKE 'Delhi';
8 Find orders between March 1 and March 15, 2024.
SELECT * FROM Orders WHERE order_date BETWEEN '2024-03-01' AND '2024-03-15';
9 Show customer names with their order amounts (JOIN).
SELECT c.name, o.amount FROM Customers c INNER JOIN Orders o ON c.cust_id = o.cust_id;
10 Show ALL customers and their orders (even if they didn't order).
SELECT c.name, o.amount FROM Customers c LEFT JOIN Orders o ON c.cust_id = o.cust_id;
11 Find total amount spent by each customer (GROUP BY).
SELECT c.name, SUM(o.amount) as total_spent FROM Customers c JOIN Orders o ON c.cust_id = o.cust_id GROUP BY c.cust_id, c.name;
12 Show only customers who spent more than 1000 total (HAVING).
SELECT c.name, SUM(o.amount) as total_spent FROM Customers c JOIN Orders o ON c.cust_id = o.cust_id GROUP BY c.cust_id, c.name HAVING SUM(o.amount) > 1000;
13 Find customers who have placed ANY order (EXISTS).
SELECT name FROM Customers c WHERE EXISTS ( SELECT 1 FROM Orders o WHERE o.cust_id = c.cust_id );
14 Find customers who spent more than the average order amount (Nested query).
SELECT c.name, o.amount FROM Customers c JOIN Orders o ON c.cust_id = o.cust_id WHERE o.amount > (SELECT AVG(amount) FROM Orders);
15 Create a view showing high value orders (amount > 1000).
CREATE VIEW HighValueOrders AS SELECT c.name, o.amount, o.order_date FROM Customers c JOIN Orders o ON c.cust_id = o.cust_id WHERE o.amount > 1000;
16 Show all orders from Delhi customers using set operations (UNION simulation).
-- Actually use JOIN, but if they ask for set ops: SELECT order_id FROM Orders WHERE cust_id IN ( SELECT cust_id FROM Customers WHERE city = 'Delhi' ); -- Or if combining two result sets: SELECT name FROM Customers WHERE city = 'Delhi' UNION SELECT name FROM Customers WHERE city = 'Mumbai';
17 Delete the Orders table completely.
DROP TABLE Orders; -- Note: Must drop child table (Orders) before parent (Customers) -- due to foreign key constraint, or use: -- DROP TABLE Orders CASCADE;
