📋 RDBMS Syntax Reference

Quick lookup for lab exam

1. Table Constraints (PK, FK)
DDL
-- Create table with Primary Key CREATE TABLE table_name ( column1 datatype PRIMARY KEY, column2 datatype, ... ); -- Create table with Foreign Key CREATE TABLE table_name ( column1 datatype PRIMARY KEY, column2 datatype, fk_column datatype, FOREIGN KEY (fk_column) REFERENCES parent_table(parent_column) );
-- Example CREATE TABLE Student ( roll_no INT PRIMARY KEY, name VARCHAR(50), class VARCHAR(20) ); CREATE TABLE Marks ( mark_id INT PRIMARY KEY, roll_no INT, subject VARCHAR(30), marks INT, FOREIGN KEY (roll_no) REFERENCES Student(roll_no) );
Remember: PK = Unique + Not Null. FK = Links to another table's PK.
2. DML Commands (INSERT, UPDATE, DELETE)
Data Manipulation
-- INSERT all columns INSERT INTO table_name VALUES (value1, value2, value3); -- INSERT specific columns INSERT INTO table_name (col1, col2) VALUES (value1, value2); -- UPDATE UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition; -- DELETE DELETE FROM table_name WHERE condition;
-- Examples INSERT INTO Student VALUES (1, 'Rahul', '10th'); INSERT INTO Student (roll_no, name) VALUES (2, 'Priya'); UPDATE Student SET class = '11th' WHERE roll_no = 1; UPDATE Student SET name = 'Ramesh', class = '12th' WHERE roll_no = 2; DELETE FROM Student WHERE roll_no = 2; DELETE FROM Student; -- Deletes ALL rows (DANGER!)
Warning: Always use WHERE with UPDATE and DELETE!
3. ALTER & DROP (DDL)
Structure Changes
-- ALTER: Add column ALTER TABLE table_name ADD column_name datatype; -- ALTER: Drop column ALTER TABLE table_name DROP COLUMN column_name; -- ALTER: Modify column ALTER TABLE table_name MODIFY column_name new_datatype; -- DROP: Delete table DROP TABLE table_name;
-- Examples ALTER TABLE Student ADD email VARCHAR(50); ALTER TABLE Student DROP COLUMN email; ALTER TABLE Student MODIFY name VARCHAR(100); DROP TABLE Student;
Tip: ALTER changes structure. DROP deletes everything. Use CASCADE to drop dependent objects.
4. ANY, ALL, EXISTS, NOT EXISTS
Subquery Operators
ANY WHERE x > ANY (subquery) True if > any value
ALL WHERE x > ALL (subquery) True if > all values
EXISTS WHERE EXISTS (subquery) True if subquery returns rows
NOT EXISTS WHERE NOT EXISTS (subquery) True if subquery empty
-- ANY: Find marks > any student in Class B SELECT * FROM Student WHERE marks > ANY ( SELECT marks FROM Student WHERE class = 'B' ); -- ALL: Find marks > all students in Class B SELECT * FROM Student WHERE marks > ALL ( SELECT marks FROM Student WHERE class = 'B' ); -- EXISTS: Find classes that have students SELECT class_name FROM Class c WHERE EXISTS ( SELECT 1 FROM Student s WHERE s.class = c.class_name ); -- NOT EXISTS: Find classes with no students SELECT class_name FROM Class c WHERE NOT EXISTS ( SELECT 1 FROM Student s WHERE s.class = c.class_name );
5. Set Operations (UNION, INTERSECT, EXCEPT)
Combining Results
-- UNION: Combine (no duplicates) SELECT column FROM table1 UNION SELECT column FROM table2; -- UNION ALL: Combine (with duplicates) SELECT column FROM table1 UNION ALL SELECT column FROM table2; -- INTERSECT: Common rows SELECT column FROM table1 INTERSECT SELECT column FROM table2; -- EXCEPT/MINUS: In table1 but not table2 SELECT column FROM table1 EXCEPT SELECT column FROM table2;
-- Examples -- Students in Class A OR Class B SELECT name FROM ClassA UNION SELECT name FROM ClassB; -- Students in BOTH Class A AND Class B SELECT name FROM ClassA INTERSECT SELECT name FROM ClassB; -- Students in Class A but NOT in Class B SELECT name FROM ClassA EXCEPT SELECT name FROM ClassB;
Rule: Both queries must have same number of columns and compatible data types.
6. LIKE & BETWEEN
Pattern Matching & Range
-- LIKE Patterns SELECT * FROM table WHERE column LIKE 'A%'; -- Starts with A SELECT * FROM table WHERE column LIKE '%a'; -- Ends with a SELECT * FROM table WHERE column LIKE '%am%'; -- Contains 'am' SELECT * FROM table WHERE column LIKE 'R_hul'; -- R + any char + hul SELECT * FROM table WHERE column LIKE '_ _ _'; -- Exactly 3 chars -- BETWEEN (inclusive) SELECT * FROM table WHERE marks BETWEEN 60 AND 80; SELECT * FROM table WHERE date BETWEEN '2024-01-01' AND '2024-12-31'; SELECT * FROM table WHERE name BETWEEN 'A' AND 'M';
Wildcards: % = any characters (0 or more), _ = exactly one character
7. Nested & Correlated Queries
Subqueries
-- NESTED QUERY (Inner runs first, once) SELECT * FROM Student WHERE marks > ( SELECT AVG(marks) FROM Student ); -- NESTED with IN SELECT * FROM Student WHERE roll_no IN ( SELECT roll_no FROM Marks WHERE marks > 80 );
-- CORRELATED QUERY (Inner runs for each outer row) -- Find students with marks above their class average SELECT name FROM Student s1 WHERE marks > ( SELECT AVG(marks) FROM Student s2 WHERE s2.class = s1.class ); -- Correlated with EXISTS SELECT name FROM Student s WHERE EXISTS ( SELECT 1 FROM Marks m WHERE m.roll_no = s.roll_no AND m.marks > 90 );
Difference: Nested = Independent. Correlated = Uses outer query data (slower, more powerful).
8. JOINS
Combining Tables
-- INNER JOIN (Only matching rows) SELECT t1.col, t2.col FROM table1 t1 INNER JOIN table2 t2 ON t1.common_col = t2.common_col; -- LEFT JOIN (All from left, matched from right) SELECT t1.col, t2.col FROM table1 t1 LEFT JOIN table2 t2 ON t1.common_col = t2.common_col; -- RIGHT JOIN (All from right, matched from left) SELECT t1.col, t2.col FROM table1 t1 RIGHT JOIN table2 t2 ON t1.common_col = t2.common_col; -- EQUI JOIN (Using = in WHERE) SELECT * FROM table1 t1, table2 t2 WHERE t1.common_col = t2.common_col;
-- Examples -- Student names with their marks SELECT s.name, m.subject, m.marks FROM Student s INNER JOIN Marks m ON s.roll_no = m.roll_no; -- All students (even if no marks) SELECT s.name, m.marks FROM Student s LEFT JOIN Marks m ON s.roll_no = m.roll_no;
Memory: LEFT = Left table is boss (all rows kept). INNER = Only matching rows.
9. Aggregate Functions
Calculations
  • COUNT(*) - Count all rows
  • COUNT(column) - Count non-null values
  • SUM(column) - Total sum
  • AVG(column) - Average value
  • MAX(column) - Maximum value
  • MIN(column) - Minimum value
-- Examples SELECT COUNT(*) FROM Student; SELECT COUNT(roll_no) FROM Student; SELECT SUM(marks) FROM Marks; SELECT AVG(marks) FROM Marks; SELECT MAX(marks) FROM Marks; SELECT MIN(marks) FROM Marks; -- Multiple aggregates SELECT COUNT(*) as total, AVG(marks) as average, MAX(marks) as highest FROM Marks;
10. GROUP BY & HAVING
Grouping Data
-- Basic GROUP BY SELECT column, aggregate_function(column) FROM table GROUP BY column; -- GROUP BY with HAVING SELECT column, aggregate_function(column) FROM table GROUP BY column HAVING condition; -- Full syntax with WHERE SELECT column, aggregate_function(column) FROM table WHERE condition GROUP BY column HAVING condition ORDER BY column;
-- Examples -- Count students per class SELECT class, COUNT(*) as total FROM Student GROUP BY class; -- Average marks per class (only if count > 5) SELECT class, AVG(marks) as avg_marks FROM Student GROUP BY class HAVING COUNT(*) > 5; -- Classes with average > 60 SELECT class, AVG(marks) as avg_marks FROM Student GROUP BY class HAVING AVG(marks) > 60;
WHERE vs HAVING: WHERE filters rows BEFORE grouping. HAVING filters groups AFTER aggregation.
11. VIEWS (CREATE, UPDATE, DELETE, DROP)
Virtual Tables
-- CREATE VIEW CREATE VIEW view_name AS SELECT columns FROM tables WHERE conditions; -- CREATE OR REPLACE VIEW CREATE OR REPLACE VIEW view_name AS SELECT columns FROM tables WHERE conditions; -- UPDATE VIEW (modifies base table) UPDATE view_name SET column = value WHERE condition; -- DELETE from VIEW (modifies base table) DELETE FROM view_name WHERE condition; -- DROP VIEW DROP VIEW view_name;
-- Examples CREATE VIEW TopStudents AS SELECT name, marks FROM Student WHERE marks > 80; CREATE OR REPLACE VIEW TopStudents AS SELECT name, marks, class FROM Student WHERE marks > 75; SELECT * FROM TopStudents; UPDATE TopStudents SET marks = 85 WHERE name = 'Rahul'; DROP VIEW TopStudents;
Remember: View is just a saved query. Updating view updates the actual table.
⚡ Quick Command Reference
  • CREATE TABLE - Make new table
  • ALTER TABLE - Change table structure
  • DROP TABLE - Delete table
  • INSERT INTO - Add data
  • UPDATE - Modify data
  • DELETE FROM - Remove data
  • SELECT - Retrieve data
  • WHERE - Filter rows
  • GROUP BY - Group rows
  • HAVING - Filter groups
  • ORDER BY - Sort results
  • JOIN - Combine tables
  • CREATE VIEW - Save query as virtual table
